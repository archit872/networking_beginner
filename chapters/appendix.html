Appendix

Pitfalls, Checklist & Progression

Quick reference for common mistakes, a structured debugging checklist, and how to progress from beginner to intermediate networking.

Top 10 Pitfalls to Avoid

  1. Confusing IP and MAC addresses.
  2. Omitting the default gateway for off-subnet traffic.
  3. Over-relying on ping as proof of application health.
  4. Misconfigured or stale DNS records.
  5. NAT conflicts between overlapping private networks.
  6. Ignoring MTU issues and blocked ICMP for PMTUD.
  7. Assuming firewall rules are permissive when they’re restrictive.
  8. Binding services to 127.0.0.1 instead of 0.0.0.0.
  9. Skipping documentation of subnets, routes, and port usage.
  10. Hardcoding fixes (/etc/hosts, “allow all” rules) instead of addressing root causes.

The Golden Debug Checklist

Use this “golden path” when diagnosing networking problems:

  1. Link: Is the interface up? Correct VLAN or Wi-Fi?
  2. IP: Correct address, mask, and gateway configured?
  3. DNS: Does name resolution return expected IPs?
  4. Routing: Does traceroute reveal path to destination?
  5. Transport: Can TCP handshake complete (SYN, SYN-ACK)?
  6. TLS: Certificates valid, SNI matches, no protocol mismatch?
  7. Application: HTTP/DNS/SMTP response as expected?
Tip: At each layer, use at least one tool (ping, nslookup, curl, Wireshark) to gather evidence before moving on.

Progression Map

Where to go after completing this beginner curriculum:

  • Beginner End Goal: Vocabulary, OSI/TCP-IP models, IP basics, diagnostic tools, build/debug small networks.
  • Intermediate Entry Point: Practical competence in patterns (NAT, VPN, routing protocols), cloud VPC design, TLS in depth, observability tooling.
  • Advanced Path: Internet-scale routing (BGP, anycast), optimization (QUIC, HTTP/3), advanced network security, and research-grade architectures.

Review Checklist Before Moving On

  • Can you explain the difference between TCP and UDP with real-world examples?
  • Can you configure IP addresses, gateways, and DNS on a small lab network?
  • Can you debug a failed website load with ping, traceroute, nslookup, and curl?
  • Have you captured packets and identified DNS, TCP, and HTTP flows?
  • Do you know common pitfalls (DNS misconfig, NAT overlap, MTU issues) and how to avoid them?
